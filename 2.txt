2.1 Риски
- API робота нестабилен
- Несовпадение статусов между роботом и WMS
- Низкая производительность сети склада
- Очередь заданий переполняется при пиковых нагрузках
- Робот возвращает неконсистентные (противоречивые, нарушающие логическую целостность) статусы
- Рассинхронизация времени на разных устройствах 
- Физические ограничения склада (маршруты, зоны)
- Отсутствие retry логики (возможна потеря заданий)
- Некорректная обработка ошибок робота


2.2 Зависимости 
- Доступность API робота
- Доступность API WMS
- Стабильность WiFi сети на складе
- SLA ответа робота
- Формат статусов робота
- Очередь сообщений (Redis / RabbitMQ)
- Временная синхронизация систем
- Доступ к тестовой среде склада
- Безопасность (VPN, firewall)
