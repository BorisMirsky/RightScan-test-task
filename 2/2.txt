# 2. Риски ⚠️

1. API робота нестабилен
2. Несовпадение статусов между роботом и WMS
3. Низкая производительность сети склада
4. Очередь заданий переполняется при пиковых нагрузках
5. Робот возвращает неконсистентные статусы
6. Изменение API WMS без уведомления
7. Проблемы синхронизации времени (timestamps)
8. Физические ограничения склада (маршруты, зоны)
9. Отсутствие retry логики → потеря заданий
10. Некорректная обработка ошибок робота

# Пример mock API (FastAPI)

```python
from fastapi import FastAPI
from pydantic import BaseModel
from enum import Enum
import uuid

app = FastAPI()

class TaskStatus(str, Enum):
    created = "created"
    in_progress = "in_progress"
    completed = "completed"
    failed = "failed"

class Task(BaseModel):
    order_id: str
    source_bin: str
    target_bin: str

tasks = {}

@app.post("/robot/task")
def create_task(task: Task):
    task_id = str(uuid.uuid4())
    tasks[task_id] = {
        "status": TaskStatus.created,
        "task": task
    }
    return {"task_id": task_id, "status": TaskStatus.created}

@app.get("/robot/task/{task_id}")
def get_status(task_id: str):
    return tasks.get(task_id, {"error": "not found"})

@app.post("/robot/task/{task_id}/complete")
def complete_task(task_id: str):
    if task_id in tasks:
        tasks[task_id]["status"] = TaskStatus.completed
    return {"task_id": task_id, "status": TaskStatus.completed}
```
